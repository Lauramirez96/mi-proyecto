7. Actualizar cambios 

 Debemos  mantener  nuestros  repositorios  local y remoto actualizados. 
Es una buena práctica, cada vez que terminamos de trabajar en alguna sección de nuestro código, o al finalizar el día de trabajo, 
subir al repositorio los cambios realizados.


para esto utilizamos los siguientes comandos: 

 a. git add . 
 b. git commit -m "inicio de proyecto"
 c. git push  
 
Si comienzan a investigar sobre los comandos de GitHub verán que hay muchísimos comandos y funcionalidades. 
Por ahora nos centraremos en estos tres: add, commit y push. 
Son los que nos permiten compartir nuestro código para que pueda ser leído desde otra computadora  y  tener guardada  una
  copia  de  seguridad  en  la  nube  para  tenerlo  disponible  en  caso  de  tener  algún problema con nuestra máquina.
